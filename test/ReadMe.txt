■この test フォルダについて

これは、なでしこが正しく動作しているかテストするためのものです。

主に開発段階でなでしこの動作を検証するためのものですが、
運用時に、なでしこの動作がどうもおかしいと感じたときに
ここにあるプログラムを実行して、動作を検証することもできます。

そのため、通常時には実行する必要はありません。

詳しくは次のページを参考にしてください。
http://nadesi.com/man/page/FAQ%252F%E3%82%A4%E3%83%B3%E3%82%B9%E3%83%88%E3%83%BC%E3%83%AB%E3%83%95%E3%82%A9%E3%83%AB%E3%83%80%E3%81%AB%E3%81%82%E3%82%8Btest%E3%83%95%E3%82%A9%E3%83%AB%E3%83%80%E3%81%AF%EF%BC%9F

■テストの概要

簡単な実行テストを行います。
テストに成功したら、結果をサーバに記録します。

テストを実行するには、「全テスト実行.nako」を実行します。


■テストを作ってくださる方へ

「AとBで実行テスト」のように書いてテストを実行します。

テストは、<test\dnako> や <test\nakooffice> などのディレクトリの下に配置します。
全テスト実行.nakoを実行すると動的に読み込まれて実行します。

※なでしこでは、同じ名前の関数を作れません。そこで、テストを実行する関数の名前は「●テスト{フォルダ名}{ユニット名}{関数名}」という形式で作ります。
